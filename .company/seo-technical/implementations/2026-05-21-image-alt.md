# 実装記録: 画像 alt属性 / aria-label 一括整備

## 背景
ベースライン監査で発覚 — KaigaiQはCSS background-imageを多用しており `<img>` 要素がほとんど存在せず alt属性が未整備。視覚障害者・画像検索クローラー双方への配慮が不十分だった。

## 方針
1. **真に画像である領域（店舗ギャラリー）** → `<img>` タグ化して alt 付与
2. **装飾＋情報の境界領域（カード背景）** → `role="img"` + `aria-label` 自動付与
3. **純粋な装飾（グラデーション・パターン）** → 触らない

## 実装

### shop.js / shop.html / shop.css
- `renderShop()` のギャラリー生成を背景画像 → `<img loading="lazy" decoding="async" alt="店名 店内写真 N枚目（city・type）">` に変更
- shop.css に `.shop-gallery-grid .gallery-item img { width:100%; height:100%; object-fit:cover; display:block }` を追加
- ヒーロー画像 `#shopHeroBg` に `role="img"` + `aria-label="{name}（{flag} {city}・{type}）のヒーロー画像"`
- 画像モーダルの拡大img に alt を動的設定（galleryDataから取得）

### script.js
新規関数 `labelBg(selector, getLabel)` を追加し、以下のクラスにaria-labelを自動付与:
- `.area-card-img.lazy-bg` → 「{flag} {国/都市名} エリアのイメージ写真」
- `.new-shop-img.lazy-bg` → 「{店舗名}（{flag} {city}・{type}）の店舗写真」
- `.job-card-img.lazy-bg` → 「{店舗名}（{flag} {city}）の求人イメージ」
- `.news-card-img.lazy-bg` → 「{記事タイトル} の記事サムネイル」

### index.html 全店舗グリッド
動的レンダリングコードに `aria-label` を埋め込み、各店舗カードに「{name} の店舗ページへ」、画像に「{name}（{flag} {city}・{type}）の店舗写真」を付与。

### news.js
`.news-featured-img` / `.news-article-img` に `role="img"` + `aria-label="[カテゴリ] 記事タイトル の記事サムネイル"`

## カバレッジ
| 対象 | 改善前 | 改善後 |
|------|--------|--------|
| shop.html ギャラリー（35店 × 平均6枚 = 約210枚） | alt なし | 全てに「店名 店内写真 N枚目」 |
| shop.html ヒーロー（35店） | aria-label なし | 全てに「店名（city・type）のヒーロー画像」 |
| index.html エリアカード（12個） | aria-label なし | 「{flag} {city} エリアのイメージ写真」 |
| index.html 新規掲載店（4個） | aria-label なし | 「{店名}（{city} ・ {type}）の店舗写真」 |
| index.html ピックアップ求人（5個） | aria-label なし | 「{店名}（{flag} {city}）の求人イメージ」 |
| index.html ショップニュースカード（4個） | aria-label なし | 「{記事タイトル} の記事サムネイル」 |
| index.html 全店舗グリッド（35個） | aria-label なし | 動的aria-label付与 |
| news.html フィーチャード+記事（10+個） | aria-label なし | 「[{カテゴリ}] {タイトル} の記事サムネイル」 |
